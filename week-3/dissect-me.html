in DBC our focus has been on learning to use GitHub as a Version Control System and making a habit of the commands required to use GitHub effectivley that will eventually allow us to work with teams of any size on larger projects. The Version Control System involves creating a temporary duplicate of the repository (the project at hand) that you can make and save changes to without corrupting the master version everyone else is working off. This is called checking out a branch. Once you check out a branch, you can create new material or change existing coding and structure and it won't change the master or anyone else's work untill your branch is merged with the master repo. As these updated branches are merged back to the master a giant timeline logs every change with a time stamp, all the changes, and the name of the user who made the changes. This timeline alone is a valuable tool for checking updates and progress but it also works like a time machine and a user can pull a branch from any spot on the timeline to backup or restore.

In version control system, a commit adds latest changes to software repository. This way when other user do any update or checkout a project, they will receive a latest version, unless they specify they wish to retrieve a previous version of the source code in the repository. What are the best practices for commit messages? A project's long-term success rests (among other things) on its maintainability, and a maintainer has few tools more powerful than his project's log. Sometimes a single line is fine, especially when the change is so simple that no further context is necessary. e.g “fixed typo” when a commit merits a bit of explanation and context, you need to write a body with Subject. e.g: Fixed bug#23 Expired user were is able to create certificate. As per business requirements, this is not allowed. What does the HEAD^ argument mean? HEAD is a reference to the last commit in the current checked out branch. HEAD~3, HEAD~4, HEAD~5, are the 3rd 4th and 5th commit from HEAD respectively.
